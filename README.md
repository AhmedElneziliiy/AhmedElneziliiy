## 🏗️ Architecture Approach

I structure my .NET projects with **Clean Architecture** — separating API controllers, core business logic, and infrastructure/data access into distinct layers. Key patterns I apply:

- **Repository Pattern + Unit of Work** — for maintainable, testable data access
- **DTOs + AutoMapper** — for clean data contracts between layers
- **JWT + ASP.NET Identity** — for secure, stateless authentication
- **SignalR** — for real-time features (notifications, chat, presence)
